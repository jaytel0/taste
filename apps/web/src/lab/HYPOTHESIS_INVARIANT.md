# Skill Lab Hypothesis Invariant

The lab does not make hypotheses about the measurement prompt.

The measurement prompt, such as "Design a dashboard.", is only a blind test surface used to
compare outputs from different versions of the same reusable skill.

Hypotheses must be about alternative ways to write `SKILL.md` from the source rule set, using the
current skill as the baseline. Valid hypothesis axes include:

- Section weighting: over-index one part of the rule set versus balancing all parts.
- Specificity: long granular rules versus short high-level principles.
- Structure: procedural checklists versus declarative constraints.
- Emphasis: anti-pattern-heavy guidance versus positive construction recipes.
- Ordering: front-loaded hierarchy versus distributed local rules.
- Explanation style: concrete examples versus abstract taste language.

Hypotheses must not mention or optimize for the current prompt, dashboards, generated HTML,
iframes, screenshots, or any single output domain.
